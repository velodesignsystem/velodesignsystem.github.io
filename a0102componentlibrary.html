Designing with components
Building all the screens necessary to facilitate an app can be a complex process, the image above gives an example image of workflow for a section of our app. A component library was created to ensure a consistent user experience thoughout the many screens of our application, help designers work and collaborate more efficiently, and facilitate consistency in developer builds. Our app has 5 different sections to our component library, outlined below.
01 - Global components
Global components refer to components that are used regardless of the background color. These components include Velo specific components like toasts, modals and menus, but also iOS and Android dependant components like keyboards, notifications and nav bars.
For anyone using our Sketch Velo Design Library, these global components can be accessed by hitting the insert button and following the path above.
02 - Icon components
Icons are used throughout our app to inform users of actions or visually represent processes our bank provides, and are a large part of the look, feel and personality of our Velo experience. The icons section is subdivided into icon context, with a “01 Global” section for icons that appear in many different contexts.
For anyone using our Sketch Velo Design Library, these icon components can be accessed by hitting the insert button and following the path above.
03 - On-Dark components
Our app is generally divided into two background types. On-Dark is a deep blue gradient background that is a core part of our brand identity. The On-Dark component section contains the components that should only be used with this kind of background. Colors of these components are optimized to meet WCAG 2.0 Compliance accessibility standards with AA grade on the dark background. Components include CTA buttons, page headers, various input fields, dashboard navigation bars and card systems.
For anyone using our Sketch Velo Design Library, these on-dark components can be accessed by hitting the insert button and following the path above.
04. On-White components
The on white background is generally used for text or form heavy pages in our app. The On-White component section contains the components that should only be used with this kind of background. Colors of these components are optimized to meet WCAG 2.0 Compliance accessibility standards with AA grade on the white background. Components include CTA buttons, page headers, various input fields,  and card systems.
For anyone using our Sketch Velo Design Library, these on-white components can be accessed by hitting the insert button and following the path above.
05 - Feed components
The feed section is a unique part of our app, created by the 17seconds design studio, where users can access newsfeed articles that are related to finance topics. It has been given its own component section since it functions a little differently from the rest of the Velo banking experience.
For anyone using our Sketch Velo Design Library, these feed components can be accessed by hitting the insert button and following the path above.
