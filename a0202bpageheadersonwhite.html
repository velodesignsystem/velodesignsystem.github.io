Page header component
Thre are less variations of the On-white headers, but there are circumstances where a 2-line format is required for the header title.

We should always try to apply the Single-line title version of the component if possible.

The header title will always be in title case, as shown above, with each word begining with a capital letter.
The header also includes navigation actions the user can take on the left and right side, indicated by the icons. The left side should always be reserved for the back nav arrow, applied if needed. Options on the right side can vary; common usages include 3 dots menu, share icon, trash can for delete and X icon for close, applied if needed.
Header application
Header component will always be anchored to the top of the page, there is never any circumstance where this will change. There may be some circumstances where the component will not have a title or any navigation icons, but there will always be a status bar.
Header - spacing
Content will always begin at least 18px below the header component. The images above show a range of different pages where the spacing rule is applied.
Header - hover scroll
Header component will always be anchored to the top of the page, there is never any circumstance where this will change. If a page length extends beyond the length of the screen, as the user scrolls down the page header will remain anchored to the top of the page, floating above the content below. A drop shadow effect fades in below the component to let the user know they are no longer at the top of the screen.

Drop shadow values
Blur: 10
Alpha: 50
Color: Black
