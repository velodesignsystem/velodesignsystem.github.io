Modals - Overview
A modal window is an element that sits on top of an applicationâ€™s main window. It creates a mode that disables the main window but keeps it visible with the modal window as a child window in front of it. Users must interact with the modal window before they can return to the parent application.

We have two types of modal sizes in our Velo app, a short modal and a tall modal, as shown above.

We use modals to confirm an action, and give users the help and information necessary for them to understand context and make the best decision.
Modals - Short
There are two types of short modal; one button and two button options. Short modals can only contain brief messages, if a message cannot be contain within the short modal size, the tall modal format should be used instead. Buttons are always placed at the bottom of the card, giving the user options for their next action, which will exit the modal view.
Modals - Tall
Tall modals are used when there is a need for more text to adequately inform the user. Tall modals have titles at the top of the modal card, and text paragraphs inside the card. Buttons are always placed at the bottom of the card, giving the user options for their next action, which will exit the modal view.
Standard tall modal
The standard tall modal is available in the varaitions above, with one or two line modal titles at the top of the modal, and one or two buttons at the bottom of the modal.
Scrollable tall modal
The scrollable tall modal is only used when there is too much information to fit within the standard tall modal. We should always strive to make messages fit within the standard tall modal, the scrollable version is used only when necessary.

The scrollable tall modal is only used for help text circumstances. Users are able to scroll down the text see the content in its entirety. The back button is anchored at the bottom of the screen, allowing the user to exit the modal once they have read the information that is relevant to them.
Modals - Screen placement
All modals should always be placed in the vertical and horizontal center of the device screen, as shown above. The simple logic helps ensure consistency of placement between different device sizes.
